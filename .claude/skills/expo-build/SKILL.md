# Expo Build

Build, prebuild, and run Expo apps.

## Quick Reference

### Dev Mode (Expo Go / dev client)

```bash
# Start Metro + Expo Go
pnpm --filter mobile-expo-financial-app start

# iOS simulator
pnpm --filter mobile-expo-financial-app ios

# Android emulator
pnpm --filter mobile-expo-financial-app android
```

### Native Builds (dev client)

```bash
# iOS simulator build
pnpm --filter mobile-expo-financial-app ios:build

# iOS physical device
pnpm --filter mobile-expo-financial-app ios:build:device

# Android emulator
pnpm --filter mobile-expo-financial-app android:build

# Android physical device
pnpm --filter mobile-expo-financial-app android:build:device
```

### Prebuild (generate native directories)

```bash
# Generate ios/ and android/ directories
pnpm --filter mobile-expo-financial-app expo prebuild

# Clean prebuild (remove and regenerate)
pnpm --filter mobile-expo-financial-app expo prebuild --clean
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Port 8081 in use | `lsof -ti:8081 \| xargs kill -9` |
| Pod install fails | `cd apps/mobile-expo/ios && pod install --repo-update` |
| Stale DerivedData | `rm -rf ~/Library/Developer/Xcode/DerivedData` |
| Simulator not found | `xcrun simctl list devices available` |
| Build error after config change | `expo prebuild --clean` |

## Key Config Files

- `apps/mobile-expo/app.json` — Expo config (bundleIdentifier, plugins, etc.)
- `apps/mobile-expo/metro.config.js` — Metro bundler config (monorepo watchFolders)
- `apps/mobile-expo/babel.config.js` — Babel config

## Gotchas

- Always close simulator between app switches (Metro port conflict)
- Only one Metro bundler at a time
- `ios/` and `android/` dirs in mobile-expo are gitignored (generated by prebuild)
- For mobile-expo-ejected, `ios/` is also gitignored
- Use `ios:sim` scripts for bare RN CLI app to avoid targeting physical devices
- Physical device deployment uses wireless debugging (see `docs/modus-operandi/iphone-wireless-deploy.md`)
