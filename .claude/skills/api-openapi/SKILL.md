# API & OpenAPI

Manage the Express REST API (apps/api) and HeyAPI HTTP client (packages/http-client).

For the full implementation plan, read `docs/plans/phase-7-api-and-http-client.md`.

## Architecture

```
apps/api/                    # Express REST API
  src/
    routes/                  # Route handlers
    middleware/               # Auth, validation, error handling
    openapi/
      spec.yaml              # OpenAPI 3.1 specification (source of truth)
  package.json

packages/http-client/        # @financial-app/http-client
  src/
    generated/               # Auto-generated by HeyAPI from spec.yaml
  package.json
```

## Workflow

### 1. Define API endpoint

Edit `apps/api/src/openapi/spec.yaml` — this is the single source of truth.

### 2. Implement route handler

Create/update route handler in `apps/api/src/routes/`.

### 3. Generate HTTP client

```bash
pnpm --filter @financial-app/http-client generate
```

This reads the OpenAPI spec and generates typed client code via HeyAPI.

### 4. Consume in apps

```ts
import { getTransactions } from '@financial-app/http-client'
```

## Validation

- [ ] OpenAPI spec is valid YAML
- [ ] All routes have corresponding handlers
- [ ] Generated client types match spec
- [ ] No hardcoded API URLs (use environment variables)
- [ ] Error responses follow consistent schema

## Gotchas

- OpenAPI spec is the source of truth — never hand-edit generated client code
- Always regenerate client after spec changes
- API credentials live in `.env` files — never in code or skills
- Use `zod` for request validation in route handlers
